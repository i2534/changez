# Changez Web UI 设计稿

> 日期: 2025-05-20

## 一、项目理解

**Changez** 是一个 Go 后端服务，追踪 AI 编码工具（OpenCode / Claude Code / Cursor / Human）对文件的变更记录。

### 核心数据模型

| 实体 | 关键字段 |
|------|---------|
| **Project** | id, name, root_path, extra, is_deleted |
| **File** | id, project_id, path, latest_version_id |
| **Version** | id, file_id, storage_mode(blob/delta/delete), action, source_id, changed_at |
| **Source** | opencode, claude-code, cursor, human |

### 现有 REST API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/api/snapshot` | 捕获文件快照 |
| GET | `/api/files?project=&limit=&offset=` | 文件列表 |
| GET | `/api/files/versions?path=<path>&source=&action=&since=&until=&limit=&offset=` | 文件版本历史 |
| GET | `/api/files/diff?path=<path>&from=<from>&to=<to>` | 两版本 unified diff |
| GET | `/api/files/restore?path=<path>&version=<versionId>` | 恢复指定版本内容 |
| GET | `/api/projects` | 项目列表 |
| POST | `/api/projects` | 创建项目 |
| DELETE | `/api/projects/{id}` | 删除项目 |
| GET | `/api/stats` | 全局统计 |
| GET | `/api/snapshots/latest?projectRoot=` | 最新快照缓存 |

## 二、UI 信息架构

```
Changez Web UI
├── ① Dashboard（首页总览）
├── ② Projects（项目列表）
├── ③ Files（文件浏览器）
├── ④ File Timeline（单文件版本时间线）
└── ⑤ Diff Viewer（双版本对比 + 内容查看）
```

## 四、技术选型建议

| 层面 | 方案 | 理由 |
|------|------|------|
| **前端框架** | React 18 + TypeScript | 生态成熟，diff 库支持好 |
| **构建** | Vite | 快，配置简单 |
| **CSS** | Tailwind CSS v3 | 快速原型，体积小 |
| **路由** | React Router v6 | SPA 路由 |
| **Diff 渲染** | `diff2html` | 直接渲染 unified diff，支持 side-by-side |
| **图标** | 内联 SVG | 零依赖 |
| **嵌入方式** | Go `embed.FS` | 单二进制部署，无需独立前端服务器 |

**刻意不用的**：图表库（Dashboard 用纯 CSS 条形条）、状态管理库（React Context 够用）

## 五、需要新增/修改的后端 API

现有 API 已覆盖大部分需求。需新增 2 个端点，修改 1 个端点：

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/ui/auth-required` | 无需 | 返回 `{ "required": bool }`，供前端判断是否需要 token |
| GET | `/api/recent-activity?limit=20` | 需要 | Dashboard 最近变更流 |
| GET | `/api/projects` | 需要 | **修改**：返回中增加 `fileCount` 字段 |

**不需要新增的**（已有替代方案）：
- `version-content` → 现有 `/api/files/{path}/restore/{versionId}` 已覆盖
- `versions/hourly` → 已取消趋势图，用变更列表替代
- `files/{path}/count` → 现有 `/api/files/{path}/versions` 已返回 `totalVersions`

## 七、总结

这个设计围绕「文件版本时间线」作为核心体验，Dashboard 提供全局概览，Projects → Files → Timeline 形成三级导航。所有数据均可通过现有 REST API 获取，只需补充 2 个新接口 + 修改 1 个接口。前端用 React + Tailwind，通过 Go embed 内嵌实现单二进制部署。
